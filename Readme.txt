本项目是爬取大麦网的资源，可以根据自己需要的关键字来爬取（关键字定义在settings文件中的KEY_WORD（可以是某个演员的名称）），默认为爬取所有的数据，并且会保存在mongodb数据库中，集合的名称以爬取的关键字命名，爬取的字段名称分别为演唱会的名称（name）、演唱会的演员：（actors）、演唱会的时间年.月.日 时:分（showtime）、演唱会举办的城市（city）

我认为本项目存在的不足：爬取的是所有的数据，没有对数据进行限制，比如举办的地点；没有爬取图片、价格区间、具体地点等数据；在spider文件中的开始url过于繁琐，但是还没有找到更好的思路；

我学到了：
1.在spider中可以不定义start_url，并且可以在start_requests中“现场塑造”自己需要的起始url，并且可以从中爬取到以后需要的数据
2.大麦网网站的特殊性：虽然是用ajax技术，但是url构成和含义很简单，并且网页所需要的数据都是从另一个相关的网页中得到的，并且另一个网页是静态页面，得到的数据是str型的，如果弄明白了网站url的含义，可以很简单的构造出所有需要的url。
3.str型的数据可以用json.loads()来转换成字典数据，前提是str数据的格式和json的数据格式相同比如括号内用的是英文双引号
